# 🏗 VANGUARD: PROJECT ARCHITECTURE (Source 03)

> **Document Type:** The "Golden Thread" Logic
> **Purpose:** Explaining how projects connect.

---

## 1. The "Golden Thread" Methodology
Traditional courses have "disconnected" projects (e.g., "Build a calculator," then "Build a weather app").
Vanguard uses a **Golden Thread**: **One single project that evolves** across all 4 modules.

### Why?
*   **Retention:** Learners care more about a project they have nurtured for weeks.
*   **Complexity:** Allows for deeper complexity because we don't restart from zero every module.
*   **Portfolio:** Results in one *massive, impressive* artifact rather than 4 small ones.

---

## 2. Project Stages (The Evolution)

### Stage 1: The Soul (Module 1)
*   **Artifact:** `system_prompt.txt`
*   **Description:** A 500-word definition of the AI's persona.
*   **Example:** "Jarvis, a witty but cautious butler who prioritizes user privacy."

### Stage 2: The Hands (Module 2)
*   **Artifact:** `agent_tools.py` (or No-Code equivalent)
*   **Description:** The functional logic.
*   **Example:** Jarvis can now access Gmail API and Google Calendar.

### Stage 3: The Face (Module 3)
*   **Artifact:** `avatar.png` + `voice_clone.wav`
*   **Description:** The sensory interface.
*   **Example:** Jarvis now has a British accent and a generated holographic face.

### Stage 4: The Law (Module 4)
*   **Artifact:** `governance_policy.pdf`
*   **Description:** The rules of engagement.
*   **Example:** "Jarvis is forbidden from deleting emails without confirmation."

---

## 3. The "Bridge" Micro-Projects
Small "palate cleansers" between major modules to prevent burnout.

*   **The Logic Puzzle (Bridge 1.5):** A pure logic game (no AI) to prepare for Agent coding.
*   **The Art Gallery (Bridge 2.5):** A pure creative session (no logic) to relax after Agent coding.
